Goals¶

The purpose of the www.chanceme.info website is allow students and parents an easy to use means predicting the probability of getting into a given school. The site allows 'what-if' scenarios so a student can determine the effect on the probability by changing her profile and actions.

We feel that this is an important service lacking to the four million college applicants every year. Students try time and money consuming activities and extracurriculars without any knowledge of the impact. Our data shows that GPA and standardized tests have a far larger impact than a third or fourth extra activity. The purpose of the website is allowing the student to ascertain this for themselves.

Additionally, it allows the student to prioritize their college applications. If a given college has only a 30% chance of acceptance, perhaps the early decision card is better spent at another school with 60% chances. The student can see the effect of early decision and whether it would make a likely impact at a given school.

Features¶

  • Reactive site, requiring no form submissions
  • Mobile friendly, works with popular mobile devices (although this has not been extensively tested)
  • Plots of candidate placement vs historical competition
  • Criteria comparison: allows the student to see relative importance of one factor vs another.

Implementation¶

As seen in the diagram at the top of this page, the website consists of three major subsystems:

  • a static HTML site (hosted at SquareSpace)
  • an interactive iframe which allows users to enter their information and see predictions and analysis (written in R / Shiny, hosted at http://www.shinyapps.io/)
  • a webservice that produces predicted probabilities for a given candidate (written in Python / Flask, hosted on OpenShift)

This implementation allowed us to parallelize development time as different people could work on different subsystems while minimizing development type by re-using code whenever possible. The prediction webservice takes the knowledge gleaned from our classification research and puts it into production using the same Scikit-learn libraries and hyperparameters.

For extensive details about the implementation and deployment, see Appendix B - Build and Deployment.
